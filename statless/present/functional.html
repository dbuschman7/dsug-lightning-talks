microservice architectural style

  • modularlarity in deployment, not just code
  • strict use of an interface
  • EXAMPLES: HTTP REST // GRPC ETC.
  • allowing for different services to be written in different programming languages

stateless servers

  • requests can be dispatched to different server nodes with no need for session data sharing
  • more scalable - more roubust - simple redundancy
  • use of mongo to hold sessions means client will send same amout of data per request as a stateful server
  • server only serves the static html // no framework magic
  • EXAMPLE: angular will build the client via service calls and hold session header/token/key
