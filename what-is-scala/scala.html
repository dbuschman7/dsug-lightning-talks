What is Scala

  • acronym for “Scalable Language”
  • It can grow with you
  • Can rely on it for large mission critical systems
  • syntax is concise and low ceremony
  • the compiler can infer types for you
  • It is a pure-bred object-oriented language.
  • It is also a full-blown functional language

Object Oriented

  • pure-bred object-oriented language
  • every value is an object
  • every operation is a method-call
  • advanced architectures through classes and traits
  • many design patterns are already natively supported
  • add new operations to existing classes, called pimping
  • implicits allow code help without getting in the way

Functional

  • full-blown functional language
  • syntax is fairly conventional
  • first-class functions
  • library with efficient immutable data structures
  • a general preference of immutability over mutation
  • a gradual, easy migration to a more functional style
  • use it as a “Java without semicolons”
  • gradually eliminate mutable state in your applications
  • Scala is not opinionated, use it with any style you prefer

Seamless Java Interop

  • runs on the JVM
  • Java and Scala classes can be freely mixed
  • the Scala compiler contains a subset of a Java compiler
  • Case matching is more powerful in Scala
  • All Java libraries are available
  • Build tools like ant or maven
  • IDEs like Eclipse, IntelliJ, or Netbeans
  • Akka, Finagle, and the Play web framework include dual APIs

Functions are Objects

  • Scala’s approach is to develop a small set of core constructs that can be combined in flexible ways.
  • This applies also to its object-oriented and functional natures.
  • Functional and Object-oriented are unified to a degree
  • The function type is just a regular class.
  • Algebraic data types found are modelled in Scala as class hierarchies
  • Pattern matching is possible over arbitrary classes

Future Proof

  • shines when it comes to scalable server software
  • makes use of concurrent and synchronous processing
  • parallel utilization of multiple cores and distributed processing
  • Its functional nature makes it easier to write safe multi-threaded code
  • typically less reliance on mutable state
  • provides powerful tools for organizing concurrent systems
